# 记忆系统架构方法论

## 核心理念

AI 记忆系统不应追求"模拟人脑"，而应追求**人脑 + 外脑（书面存档）的混合架构**——这是人类几千年文明实践的进化稳定策略。

## 三层架构

```
┌─────────────────────────────────────┐
│         MEMORY.md / USER.md          │  桌面笔记本
│   (3-5K chars, 随会话注入 system prompt) │  最快拿到，常翻，必须精简
│       缓存命中成本 ≈ $0               │
├─────────────────────────────────────┤
│            details/ 主目录             │  图书馆索引台
│  (结构化 markdown, 按主题分类存储)      │  告诉你去哪找
│            read_file 读取              │
├─────────────────────────────────────┤
│          details/archive/             │  书房书架
│    (全文保存, 按归档批次分段)           │  全本原文，grep 秒定位
│         grep/rg 全文检索               │
└─────────────────────────────────────┘
```

## 三条规则

### 规则一：memory 只写索引，不写详情

MEMORY.md 和 USER.md 是"目录"不是"百科全书"。内容：
- 系统规则（我怎么工作）
- 用户偏好（你喜欢什么）
- 指向 details/ 的短索引行

**不写**：事件详细记录、过时状态、大段描述。

### 规则二：全文永不丢失

任何从 memory 移除的信息，原文完整保存到 archive/，附带：
- 时间戳
- 主题标签
- 搜索关键词（便于 grep 定位）

"归档"不是"删除"，是"从桌面移到书架"。

### 规则三：渐进精炼（睡眠模型）

参考人脑 NREM 睡眠中的去甲肾上腺素振荡机制，定时做：

1. **低频脉冲式扫描**（每周一次，类似深度睡眠周期）
2. **识别过时事件**（14 天前的单次事件）
3. **全文归档到书架**（不压缩不摘要）
4. **更新检索索引**（追加 README.md）

不做实时监控，不做每次对话后整理——效率/效果比不划算。

## 为什么选磁盘缓存架构的模型

DeepSeek V4 Flash 的架构（滑动窗口注意力 + 磁盘 KV 缓存）恰好匹配这个记忆系统：

| 特性 | 效果 |
|------|------|
| 磁盘缓存 TTL 小时-天 | memory 再长也是缓存命中价 |
| 缓存命中 vs 未命中差 50 倍 | 同会话内连续对话成本 ≈ 0 |
| 1M 上下文窗口 | 一个会话用一周不用换 |
| Sliding Window Attention | 天然支持缓存前缀单元，可以写到磁盘 |

其他模型的全局注意力架构做不到这种磁盘缓存——不是策略问题，是架构限制。

## 对比人脑记忆

| 人脑 | 我们的系统 |
|------|-----------|
| 工作记忆（~7项） | 上下文窗口（1M tokens） |
| 海马体索引 → 新皮层存储 | MEMORY.md 索引 → details/ 文件 |
| NREM 睡眠去甲肾上腺素振荡清洗 | cronjob 归档过时事件 |
| 睡眠巩固短期→长期 | 从 memory 移到 archive（桌面→书架） |
| 程序性记忆（怎么做） | Skill 系统 |
| 情景记忆（具体事件） | 会话历史 + details/ 档案 |
| **联想触发** | ❌ 无原生机制，需 full-text search |
| **再巩固**（回想时自动更新） | ❌ memory 静态，需手动编辑 |
| **遗忘→提纯**（丢噪音保模式） | ❌ 有归档但无自动摘要提纯 |

## 实用操作

### 日常维护
- 有新的系统规则 → 写进 MEMORY.md
- 有新的用户偏好 → 写进 USER.md
- 有项目/事件的完整记录 → 写进 details/*.md
- 在 memory 里只写一行索引：`xxx -> details/xxx.md`

### 找回旧信息
```bash
grep -r "关键词" ~/details/
grep -r "关键词" ~/details/archive/
```

### 架构依据
- 2025 Cell 论文：Norepinephrine-mediated slow vasomotion drives glymphatic clearance during sleep
- 大脑在 NREM 睡眠中每 ~50 秒一个去甲肾上腺素振荡周期驱动脑脊液冲刷废物
- 启示：定期低频脉冲式整理优于连续实时监控，且整理和"工作"应该分离
